# Serve the AI-discovery surface as Markdown `llms.txt`, not JSON

The site's machine-facing surface is `/ai-index.json` (schemaVersion "2"), assembled in
`src/ai-discovery/ai-index.njk`. Two problems pushed a redesign rather than a patch. First, that
template re-flattens `course.material` by hand — a duplicate of `materialHelpers.flattenItems` and
the last **Material** seam leak (Do-not #2). Second, and more fundamental, the *format* was chosen
before its consumer was: a JSON dump implies an autonomous agent parsing a schema, but the actual
consumer we are designing for is **an LLM carrying a web/fetch tool, hitting a single URL** — for
which JSON is the wrong shape.

We move the surface to the **`llms.txt` Markdown convention**: a Markdown index plus a full-corpus
companion, rather than a JSON dump. Concretely (full decision list in
`.agents/plans/ai-discovery-format.md`):

- `/llms.txt` — a Markdown index: each **Course** with a summary and links to its **Course Pages**
  (`/courses/{slug}/{type}/`), plus a top-of-file preamble pointing at the full file.
- `/llms-full.txt` — the whole corpus inlined as Markdown: **Doc** prose *and* the extracted
  `pdfText` of **Material** PDFs. This makes it a strict Markdown **superset** of the old
  `ai-index.json`, so the JSON surface is **retired**, not kept in parallel.
- Per-language, mirroring site routing: zh-TW at the project root, en-US under `/en-US/`, each file
  monolingual (no `---zh-TW---` interleaving), generated by 11ty pagination over the two locales.
- Both generators consume `materialHelpers` — closing the Do-not #2 leak is an acceptance criterion,
  so the new surface does not re-introduce the hand-rolled flatten it replaces.

The decision rests on a 2026 convention, not a local preference. `llms.txt` is an established
agent-readable standard (Stripe, Vercel, Cloudflare, Anthropic ship one), and as of 2026-05 Chrome
Lighthouse's "Agentic Browsing" category audits for a root `/llms.txt`. For an LLM fetch tool,
Markdown is materially cheaper than HTML or JSON to ingest (reports of ~10x token reduction vs HTML;
JSON spends much of its budget on structural punctuation the model must parse but a human-readable
narrative does not).

Alternatives considered:

- **Patch the JSON in place (the original Candidate D)** — route `ai-index.njk` through
  `materialHelpers` and keep emitting JSON. Fixes the leak but leaves the format mismatched to the
  consumer; rejected once the consumer was pinned to the fetch-tool model.
- **schema.org / JSON-LD for autonomous crawlers** — serves a *different* consumer (model B,
  autonomous schema-driven agents) we explicitly did not choose. Speculative for this site.
- **HTML "Guide for AI Tools"** (the existing `/for-ai/` page) — the user's first instinct. HTML is
  noisy for LLM ingestion and inverts the convention; its only edge, browser dual-use, is already
  covered by the human-facing HTML Course Pages the index links to. `/for-ai/` is removed and its
  essentials fold into the `llms.txt` preamble.
- **Per-page `.md` variants now** (`/courses/{slug}/{type}.md`) — the scalable end-state, but
  building it before `llms-full.txt` outgrows a single fetch is speculative (CLAUDE.md "no
  speculative features"). Deferred; when it ships, the `/llms.txt` links flip from `/{type}/` to
  `/{type}.md` with no structural change.

## Consequences

- **Subpath, not domain root.** As a GitHub Pages *project* site, the canonical path is
  `/course-site-starter/llms.txt`, not the true domain root the convention and the Lighthouse audit
  expect — that root belongs to a user-site repo this project does not own. A root-targeted audit
  will miss, but any agent handed the project URL works. Satisfying the literal convention needs a
  custom domain (infra, out of scope). All internal links keep the `| url` base-path filter
  (Do-not #1), consistent with ADR-0003.
- **`llms-full.txt` grows with scanned PDFs.** Inlining `pdfText` maximises completeness but makes
  size scale with Material; the trigger for the deferred per-page `.md` split arrives earlier. That
  trade is deliberate — completeness now, split when a single fetch stops being viable.
- **`/ai-guide/` is repointed, `/for-ai/` removed.** The teacher-facing guide changes its
  instruction from "upload `ai-index.json`" to "give `llms-full.txt`"; the machine-facing
  `/for-ai/` page is deleted (its role is taken by `llms.txt` itself).
- **One new published contract to keep current.** Like ADR-0002's output gate, the generators carry
  standing maintenance: the per-Course-Page link set and the inlined field list must track the site
  as it grows.
